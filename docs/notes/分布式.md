### 应对微服务分布式计算的解决方案思考
```javascript
两种方案:
    1. 从路由服务（网关）入手。通过路由服务（proxy）把资源预先计算分配好, 并通过传参的方式，调用指定服务器上的计算服务。 --- 这种方案未证实是否可行。
    2. 从计算服务自身入手。结合redis (set not exists)特性 实现资源自分配。启动计算服务集群时，假设服务调用的同一个redis。每个计算服务都要抢占计算资源。对于某个当前的计算服务，如果想要抢占的资源抢占失败（redis set not exists failed）。那么就需要重新抢占剩下的计算资源。直至没有资源或抢占成功为止。
```
