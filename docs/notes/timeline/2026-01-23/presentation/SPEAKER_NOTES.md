### 幻灯片 1.9-1.10：核心原则

**讲解**：
Vibe Coding 有几个核心原则，我一一讲：

**原则一：人负责决策**
- AI 做什么它不应该做？业务取舍、架构拍板、上线决策。
- 关键是：**所有系统后果，必须由人负责**。
- AI 是工具，不是替代品。

**原则二：Prompt 是新代码**
- 你怎么和 AI 沟通，就变得像写代码一样重要。
- Prompt 不是随便问问题，而是**规格说明书**。
- 一个好的 Prompt 包含：上下文、约束、输出要求。

**原则三：质量回路**
[读出步骤]
1. 人表达意图
2. AI 生成方案
3. 人审查迭代

这是一个反复循环的过程。

**原则四：充分的上下文**
- AI 的能力和你提供的上下文质量成正比。
- 上下文越丰富，AI 的输出质量就越高。
- 这也是 Claude Code 这个工具存在的主要理由。

#### 幻灯片 3.1.2-3.1.3：/commit Skill 深入

**讲解**：
让我们深入看一个具体的 Skill：**`/commit`**。

**问题背景**：
- 手动写 commit message 既费时又容易不规范
- 看看传统流程：
  - `git status` — 看一眼
  - `git diff` — 看改了什么
  - `git log` — 看一下历史风格
  - 然后自己思考…… — 30 秒
  - `git commit -m "fix something"` — 太随意了

**这样的问题是什么呢？**
- 格式不一致 — 有的用 Conventional Commits，有的随意写
- 信息不清楚 — "fix bug" 和具体修了什么没关系
- 历史记录难以追踪 — 6 个月后再看代码，你都不知道为什么改的

[停顿 2 秒]

**Claude Code 的 /commit 做什么？**

[逐步讲解工作流]
1. **分析 git status** — 了解哪些文件被修改
2. **查看 git diff** — 理解具体改动内容
3. **检查 git log** — 学习项目的 commit 风格
4. **生成规范 message** — 遵循 Conventional Commits
5. **创建提交** — 规范且清晰的历史记录

**实际演示**：
假设你修改了认证相关的代码，你只需要：
```bash
$ git add .
$ /commit
```

AI 会自动生成：
```
feat(auth): Add OAuth2 integration with GitHub
 5 files changed, 250 insertions(+), 45 deletions(-)
 - Added GitHubStrategy OAuth provider
 - Configured JWT token generation
 - Updated user session middleware
 - Added security tests
```

**质量提升**：
- ✓ **格式规范** — 自动使用 Conventional Commits
- ✓ **信息清晰** — 自动总结改动要点
- ✓ **团队一致** — 学习项目历史风格，自动统一
- ✓ **可追踪** — 便于后期搜索和分析

[停顿 2 秒]

### Part 3.2：Agents（8-10 分钟）

#### 幻灯片 3.2.1：什么是 Agents？

**讲解**：
现在我们看第二个核心能力：**Agents**。

Agents 和 Skills 的区别在于：**Agents 有自主思考能力**。

[逐点讲解]
- **能思考**：不只执行指令，而是分析和规划
- **能探索**：自动发现代码库结构和上下文
- **能设计**：生成详细的实现计划
- **能执行**：完整地实现设计方案
- **有安全等级**：从只读到完全权限的分级

这很重要，因为它体现了 **人机协作的精神**。

#### 幻灯片 3.2.2：Agents 的三个关键特征

**讲解**：
Agents 有三个关键特征：

**特征一：多步骤推理**
- 不是一个 prompt 来回就完了
- 而是多轮思考和验证
- "我应该先查什么？" → 查了之后 → "接下来查什么？"

**特征二：工具使用**
- 能够使用 Read、Grep、Bash 等工具收集信息
- 不只是聊天，而是真正地和代码库交互

**特征三：自主决策**
- 基于收集的信息做出智能决策
- "这个问题需要先看什么文件"
- "这个错误的根本原因可能是什么"

[停顿 1 秒]

#### 幻灯片 3.2.3：Agent 的类型

**讲解**：
Claude Code 中有三种 Agent，它们有不同的权限级别：

[从下往上讲]

**最底层：Explore Agent**
- **只读操作**
- 能够：查找文件、搜索代码、阅读文件
- 不能：修改文件、执行命令
- 适用场景：新成员快速了解代码库、自动化分析

**中层：Plan Agent**
- **设计阶段**
- 能够：分析代码、设计方案、生成计划
- 但需要**人工批准**才能执行
- 适用场景：大的功能开发、架构重构

**顶层：General Purpose Agent**
- **完整权限**
- 能够：一切操作
- 需要更多的信任和充分的上下文
- 适用场景：完整的开发流程

[停顿 1 秒]

#### 幻灯片 3.2.4：Explore Agent 深入

**讲解**：
让我详细讲 **Explore Agent**，因为它是最容易理解、最实用的。

**什么时候用 Explore？**
当你有一个问题但不知道从哪里找答案：
- "我们的认证系统是怎么实现的？"
- "哪里处理用户上传的文件？"
- "API 错误处理的逻辑是什么？"
- "数据库连接在哪里配置？"

**Explore 能做什么？**
- 只读操作
- 在 30 秒内快速探索代码库

**它的工作流是**：
1. Glob 搜索相关文件 — "找找有没有 auth 的文件"
2. Grep 查找关键代码 — "搜索 'authenticate' 这个词"
3. Read 理解实现细节 — "读这个文件的全部内容"
4. 生成详细报告 — "总结一下流程图和代码示例"

**真实场景**：

假设你是一个新员工，第一天进来，你问：
"认证系统如何实现的？我需要理解它的工作流"

**Explore Agent 会自动做**：
1. 搜索 `src/auth/*` `src/middleware/auth.*`
2. 搜索 "login"、"authenticate"、"session"
3. 阅读核心文件
4. 追踪 JWT、bcrypt、passport 的使用
5. 输出完整的认证流程图和代码示例

**耗时：3-5 分钟**

对比传统方式，一个新员工要理解认证系统，通常需要：
- 阅读代码：30 分钟
- 询问同事：30 分钟
- 自己理解：30 分钟
- 总计：90 分钟

[停顿 2 秒]

这就是 Explore 的威力 — **快速理解陌生代码库**。

#### 幻灯片 3.2.5：Plan Agent 深入

**讲解**：
**Plan Agent** 是用来做架构设计和规划的。

**什么时候用 Plan？**
当你知道要做什么，但不确定怎么做最好：
- "我要实现缓存系统，应该怎么架构？"
- "重构这个模块的最佳方式是什么？"
- "如何添加实时通知功能？"

#### 幻灯片 3.2.7：Agent vs SubAgent

**讲解**：
让我解释一下 Agent 和 SubAgent 的区别：

**Agent（主 Agent）**：
- 由用户直接启动

**SubAgent（子 Agent）**：
- 由 Agent 自动启动
- 负责特定领域的专项工作

- 完成后返回结果给主 Agent
- 例如：性能审查 SubAgent、安全审查 SubAgent
