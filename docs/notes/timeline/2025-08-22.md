* 排查原因：系统卡顿
* 排查步骤：
    * 查看服务器监控    
      - 发现内存和硬盘之间的*io*很高
    * 登录数据库从库服务器
    * 运行 *top*
      - 发现cpu很高(超过100%) 
    * 运行 *pidstat -d 1 5* 查找io大户
    * 运行 *mongotop 5 -u uname -p password --authenticationDatabase admin*
      - 发现某些表的读取速度很慢
    * 登录从库，运行慢查询日志
    ```shell
    db.setProfilingLevel(2) // 设置日志级别
    db.system.profile.find({"ns" : {$nin:["test.oplog.rs","test.system.profile"]},}).limit(5).sort( { ts : -1 } ).pretty()
    ```
     - 发现某些语句的 *docsExamined* 很高，但是 *keysExamined* 很小。说明没有走索引。
